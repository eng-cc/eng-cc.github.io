elementary 下安装搜狗输入法

发表于 2016-05-15   |   分类于 other

elementary是基于ubuntu的linux发行版，因为它及其漂亮的界面我第一次见它就深深地喜欢上了，并把elementary作为了常用的系统，没有特殊情况已经基本不打开我的windows了，安装配置过程和ubuntu基本一样，比较坑的就是中文输入了。

先添加fcitx源

安装fcitx输入法

用im-config选择默认输入法

可能需要先安装im-config
控制台输入

danzaiNAME=”elementary OS”
VERSION=”0.3.2 Freya”
ID=”elementary OS”
ID_LIKE=ubuntu
PRETTY_NAME=”elementary OS Freya”
VERSION_ID=”0.3.2”
HOME_URL=”http://elementary.io/“
SUPPORT_URL=”http://elementary.io/support/“
BUG_REPORT_URL=”https://bugs.launchpad.net/elementary/+filebug“
```
ID=”elementary OS” 会导致搜狗输入法安装失败还可能导致apt-get不能用，改为 ID=”ubuntu”即可正常安装，重启系统即可输入中文。

发表于 2016-05-15   |   分类于 programming

前段时间有个设计的小伙伴给我推荐了好多日本的网站，个个炫酷到飞起，身为重度前端职业病患者当然会审查元素看一下都是怎么实现的了，看完总结出：日本的网速领先国内好几倍（很多背景是插的mp4……），当然也有一部分是canvas做的，比如这个时钟效果。之前就对canvas感兴趣并了解过一点，看过这些网站更坚定了去认真学一下的决心，同时也记了些笔记。

定义canvas绘制环境

线条API

线条属性
lineCap

context.lineCap=’’,线段的端点样式（线段的帽子）
三个可选属性：butt(默认),round(圆角),square(方角)；
tound和square会比默认长出一点。

lineJoin

context.lineJoin=’’,线与线相交处样式
三个可选属性：bevel(斜接)，round（圆角），miter（默认，尖角）；

meterLimit

context.meterLimit=number,
context.lineJoin=’miter’时，实际角与外边角的距离；

图形变换

位移：
用过 translate(x,y) 后接下来再使用translate时会在上次(x,y)的基础上进行位移，所以要在使用后紧跟一句translate(-x,-y)来解决。api解决办法是将每次位移变换放在 save 和 restore 函数中间

旋转：rotate（deg）,
缩放：scale（sx, xy）

图形变换矩阵

transform（a, b, c, d ,e, ,f）

a:水平缩放
b:水平倾斜
c:垂直倾斜
d:垂直缩放
e:水平位移
f:垂直位移

文字渲染

fillText和strokeText都有第四个可选参数(string ,x,y，maxlen)最大宽度，单位为px，如果有第四个参数文字会被绘制在限定宽度内。context.fillText(string,200,200,400);

font：

最多传入五个参数，context.font = ‘font-style，font-variant,font-weight,font-size,font-family’。
font-style: normal(default),italic(斜体字)，oblique（倾斜字体）后两者效果基本一样。
font-varant: normal(default),small-caps(只用于英文，小型的大写字母).
font-weigth: lighter(和normal效果一样)，normal(default)，bold(加粗)，bolder（和bold效果一样）。
font-size: 可用px，em，30%这些css方法设置，也有文字属性值，不推荐使用。
font-family: 类似css设置多种字体备选，支持@font-face。

webpack+vue+vue-router构建大型单页应用（一）

发表于 2016-05-12

前言

前些日子被boss拉去做一个学校的管理系统，静态页面是已经写好的，看了一下文件目录（一脸蒙逼）

竟然有35个页面之多，而且许多页面里都有大量的表单，数据量相当大，刚好近期工作室另一个项目用了vue的技术栈,所以也想用vue来构建一个比较大型的单页应用。
至于为啥选这套技术栈主要原因有两个吧，一是比较轻、方便操作，还有就是vue双向数据绑定比较适合项目实际。在接下来的文章中我会从头跟大家分享项目过程中一些比较重要的东西和遇到的一些问题。

脚手架

之前一直都在用gulp作为脚手架工具进行检查编译压缩等任务，喜欢它的流的概念，所以最外层还是决定用gulp来搭，下面详细说下用到的一些工具。

  • gulp-webpack
    webpack的gulp版，语法和webpack基本一致，配置好文件入口和出口就可以

  • vue-loader
    webpack的vue组件加载器，也是实现vue单页应用的核心工具，用各loader把vue的组件打包成js可以调用的模块。详见 vue-loader GitBook

  • gulp-uglify gulp-minify
    用的比较多的代码丑化和压缩工具

  • gulp-md5
    为打包压缩的文件加md5戳的工具，方便生产环境下控制缓存

同时还引入了 jade less vue vue-router 等 package。完整的依赖也列一下吧

以后会参考vue-cli 中 webpack的例子加入代码检查和测试工具并进一步完善。
